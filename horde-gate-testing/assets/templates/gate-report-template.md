## Template Usage Guide

### Required Placeholders

| Placeholder | Description | Example Value |
|-------------|-------------|---------------|
| `{{PHASE_FROM}}` | Name of phase being tested | Phase 2: UserService Core |
| `{{PHASE_TO}}` | Name of next phase | Phase 3: AuthService Integration |
| `{{STATUS}}` | Gate status | PASS / WARN / FAIL |
| `{{DECISION_CODE}}` | Numeric decision code | 0 (PASS), 1 (WARN), 2 (FAIL) |
| `{{TEST_DATE}}` | ISO 8601 timestamp | 2026-02-04T10:30:00Z |
| `{{TEST_DURATION}}` | Test execution time | 4.2s |
| `{{TESTS_PASSED}}` | Number of tests passed | 12 |
| `{{TESTS_TOTAL}}` | Total number of tests | 12 |
| `{{PASS_PERCENTAGE}}` | Pass percentage | 100 |

### Optional Placeholders

| Placeholder | Description | Used When |
|-------------|-------------|-----------|
| `{{RISK_DETAILS}}` | Detailed risk descriptions | Risks identified |
| `{{IMPLEMENTATION_NOTES}}` | Notes for next phase | WARN or PASS status |
| `{{MOCKS_USED}}` | Table of mocks | Mocks were used |
| `{{CODE_COVERAGE}}` | Coverage metrics | Coverage data available |
| `{{NEXT_STEPS}}` | Next actions | FAIL or WARN status |

### Status Values

**Gate Status:**
- `PASS` - All tests pass, proceed with confidence
- `WARN` - Tests pass but risks identified, proceed with caution
- `FAIL` - Tests failed, do not proceed

**Decision Codes:**
- `0` - PASS
- `1` - WARN
- `2` - FAIL

**Role Status:**
- `PASSED` / `FAILED` / `WARNING` / `BLOCKED` / `N/A`
